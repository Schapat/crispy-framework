# Intake

Store input snapshots here when a user prompt, attached spec, agent intake run,
or other project-start output should remain available as source material.

Intake snapshots are not living product truth. Treat them as immutable-ish
records of what was supplied at a point in time, then derive current product
contracts into `docs/product/`, story packets into `docs/stories/`, proof
expectations into `docs/TEST_MATRIX.md`, and durable rationale into
`docs/decisions/`.
