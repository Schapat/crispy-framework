# Agent Configuration

Use this folder for target-project details that agents need repeatedly but that
do not belong in product docs, story packets, or decisions.

These files are intentionally light. Adapt them after install to match the
project's actual tooling and team habits.

## Files

- `issue-tracker.md`: external issue tracker, labels, and linking rules.
- `work-states.md`: local story states, readiness states, and escalation rules.
- `domain-docs.md`: where domain language and product terminology should live.

## Update Rule

When an agent discovers a repeated project-specific convention, command, state,
or tracker rule, record it here or in the more specific artifact that owns the
truth. Do not bury recurring operating rules only in chat.
