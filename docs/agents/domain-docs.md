# Domain Docs

Use this file to tell agents where product language lives.

## Domain Language Source

Default:

- Put project-specific terms in `docs/product/glossary.md` when the project has
  accepted domain language.
- Keep process terms in `docs/GLOSSARY.md`.
- Record architecture or product decisions in `docs/decisions/`.

## Rules

- Define domain terms before using them as durable architecture names.
- Prefer product language over implementation slang when naming product
  contracts, stories, assertions, and validation rows.
- Do not use this file as a product spec. Link to the product docs that own the
  accepted behavior.
